# Insight-Writer

라스트 기록을 사라지지 않게 만드는 인사이트 아카이브 & 글쓰기 변환 스킬.

트랜스크립트, 메모, 북클럽 후기, 기억하고 싶은 한 문장 같은 **날것의 기록**을 받아서 →
인사이트를 추출하고, 아카이브로 정리하고, 매체별(블로그 · LinkedIn · 짧은 글 · 개인 아카이브) 글감으로 분류·정제합니다.

> The primary goal is not to summarize everything.
> The primary goal is **to** prevent valuable thoughts, emotions, and insights from disappearing.

이 스킬은 요약기가 아닙니다. 감정의 진실을 보존하면서 더 깊은 의미를 끌어내고, 사용자의 자연스러운 목소리를 지키는 것이 목적입니다.

---

## 핵심 특징

- **원석 보존 우선** — 다듬기 전에 실제로 한 말, 일어난 장면, 남은 감정부터 보존합니다.
- **입력 크기별 응답 스케일링** — 한 줄 입력에 10개 섹션을 쏟아내지 않습니다. 입력 분량에 맞춰 응답 깊이를 자동 조절합니다.
- **6 페르소나 내부 검토** — 6개의 분석 렌즈를 거치되, 페르소나는 **출력에 드러나지 않는 내부 레이어**입니다. 사용자는 합쳐진 결과물만 봅니다.
- **매체별 글감 분류** — 같은 기록을 블로그 / LinkedIn / 짧은 글 / 개인 아카이브용으로 각각 다르게 정제합니다.
- **프라이버시 우선** — 타인은 기본 익명화, 민감 정보는 Low/Medium/High로 분류하고 공개용으로 재프레이밍합니다.
- **목소리 보존** — "많은 생각을 하게 되었다" 같은 클리셰와 과잉 정제된 AI 톤을 피하고, 사용자의 불확실성과 모순을 그대로 살립니다.

---

## 6 페르소나 (내부 검토 렌즈)

각 페르소나는 출력에 이름이 노출되지 않습니다. 워크플로우 단계마다 해당 렌즈가 조용히 적용됩니다.

| 페르소나 | 역할 | 적용 단계 |
|----------|------|-----------|
| The Archivist | 변환 전 원본·기억나는 문장 보존 | Source Map, Memory Fragments |
| The Insight Curator | 핵심 긴장과 더 깊은 의미 추출 | Insight Bank |
| The Channel Strategist | 최적 발행 매체로 라우팅 | Writing Seed Classification, Channel Recommendation |
| The Editor | 구조·훅·서사 흐름 강화 | Draft or Archive |
| The Privacy and Risk Reviewer | 식별 정보·민감 맥락 보호 | Privacy and Risk Check |
| The User Voice Keeper | 사용자의 자연스러운 톤 유지 | Draft or Archive |

---

## 매체별 가이드

- **Blog** — 감정의 결, 모순, 풀리지 않은 질문, 서사가 필요한 이야기
- **LinkedIn** — 일·AI·리더십·커뮤니케이션과 연결된 전문적 인사이트
- **Short post** — 맥락이 거의 필요 없는, 기억에 남는 한 문장
- **Private archive** — 날것의 감정, 실명, 비판, 아직 덜 익은 생각
- **Essay seed** — 강하지만 아직 미완성인 아이디어

## 사용법

날것의 기록을 주고 정리를 요청하면 스킬이 자동으로 작동합니다.

**트리거 예시**

- "북클럽 후기 정리해서 블로그랑 LinkedIn 글감으로 나눠줘"
- "이 문장 아카이브에 남기고 싶어: _좋은 대화는 지나간 뒤에야 내 안에서 문장이 된다._"
- "오늘 미팅 메모 정리하고 공개해도 되는지 위험 점검해줘"
- "이 긴 회고를 매체별로 쪼개줘"

**입력 → 출력 예시 (짧은 입력)**

```
입력: "AI는 시작을 도와줬지만, 끝까지 가게 한 건 목적이었다."

출력:
- 핵심 인사이트 1~2개 + 왜 이 문장이 남았는지
- 추천 매체 (Short post / LinkedIn)
- 필요 시 짧은 샘플 초안
```

긴 회고를 넣으면 중심 긴장 분석 → 글감 분해 → 매체별 추천 → 공개 위험 점검 → 매체별 초안까지 전체 파이프라인이 돌아갑니다.

## 품질 기준

모든 출력은 다음을 만족해야 합니다.

- 원래의 감정적 진실이 보존되는가
- 단순 요약보다 인사이트가 더 날카로운가
- 사용자의 목소리처럼 들리는가
- 사적인 내용이 보호되는가
- 기억할 만한 문장이 최소 하나 있는가
- 응답 깊이가 입력 크기와 맞는가
- 페르소나 라벨이 출력에 노출되지 않았는가
